## 2. Tiered Lookup

```
┌─────────────────────────────────────────────────────────────┐
│                    Tiered Lookup Pipeline                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  User enters product title                                  │
│  │                                                          │
│  ▼                                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  L1: Local static match?                            │   │
│  │  ├─ YES → Autofill from local dictionary (instant)  │   │
│  │  └─ NO ↓                                            │   │
│  └─────────────────────────────────────────────────────┘   │
│  │                                                          │
│  ▼                                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  L2: Cloudflare AI Gateway cache?                   │   │
│  │  ├─ HIT → Autofill from edge cache (50-150ms)       │   │
│  │  └─ MISS ↓                                          │   │
│  └─────────────────────────────────────────────────────┘   │
│  │                                                          │
│  ▼                                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  L3: Run LLM                                        │   │
│  │  → Save response to cache                           │   │
│  │  → Autofill & let user adjust (2s)                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘

1. L1: Local dictionary — instant, offline, zero cost
2. L2: Cloudflare AI Gateway — exact cache at edge
3. L3: LLM fallback — unique items, cached once
```

## 3. Storage Mapping

**Categories & tags** → `form.data`:
```json
{ "cat":"food", "tags":["beverage","hot"], "p":"3.50",
  "o":{ "s":["Small","Medium","Large"] } }
```

**Variants** → `matter` rows:
- `variant=0` Small, `variant=1` Medium, `variant=2` Large

**Modifiers** → `bond` links:
1. Create modifier in `form`, `data={"mod":1}`
2. Create price point in `matter`
3. Link: `src=prod_latte, tgt=prod_espresso_shot, type='modifier_of'`

## 4. Semantic Cache

Exact cache = letter-for-letter. Semantic cache = similar meaning.

| Query | Exact | Semantic |
|:------|:------|:---------|
| "Pepperoni Pizza" | MISS | MISS (first) |
| "pepperoni pizza" | MISS | **HIT** (99%) |
| "Pizza with Pepperoni" | MISS | **HIT** (95%) |
| "Clasic Peperoni Pizza" | MISS | **HIT** (91%) |

### Implementation

```javascript
// 1. Embed query
const embedding = await env.AI.run('@cf/baai/bge-small-en-v1.5', { text: query });

// 2. Vector search in Vectorize
const results = await env.VECTORIZE.query(embedding.data[0], { topK: 1 });

// 3. Cosine > 0.90 = HIT
if (results.matches[0]?.score > 0.90) {
  return await env.KV.get(`cache:${results.matches[0].id}`, 'json');
}

// 4. MISS → call LLM → save
const response = await callLLM(query);
await env.VECTORIZE.insert([{ id: queryId, values: embedding.data[0] }]);
await env.KV.put(`cache:${queryId}`, JSON.stringify(response));
return response;
```
