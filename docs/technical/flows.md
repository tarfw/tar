# Database Action Flow Matrix

This matrix documents the database mutation actions triggered from the Workspace Screen ([workspace.tsx](file:///c:/tarfwk/tar/tarapp/app/workspace.tsx)). It details the routing destination‚Äîeither the local-only **Private DB** (`user_private_${self_id}.db`) or the cloud-synced **Collaboration DB** (`user_sync_${self_id}.db`)‚Äîand explicitly breaks down which part of the data flows into which SQLite table.

| Action Name / Flow | Scope Routing Logic | Target SQLite Tables breakdown | Description & Field Mapping |
| :--- | :--- | :--- | :--- |
| **`saveCustomer`** | **Collaboration DB** | <ul><li>**`matter`**: Upserted record (stores ID, code, type, owner, title, and metadata JSON stringified in `data`).</li></ul> | Creates or updates workspace entities (Customer, Business, Team Member, etc.). Syncs to vector store. |
| **`createStock`** | **Collaboration DB** | <ul><li>**`mass`**: Stores stock metadata (ID, warehouse parent ID, type = stock, quantity, unit value, and `{name}` in JSON `data` ‚Äî the authoritative item name).</li><li>**`motion`**: Appends initial audit record (opcode `406` - TRANSFER_IN) capturing the initial quantity delta; payload carries only `{src}`, not a copy of the name.</li></ul> | Adds SCM stock item inside a warehouse with an initial quantity audit. |
| **`startStockTransfer`** | **Collaboration DB** | <ul><li>**`mass`**: Updates `qty` value (decreases for OUT, increases for IN).</li><li>**`motion`**: Appends audit record (opcode `405` - TRANSFER_OUT or `406` - TRANSFER_IN) capturing the delta and timestamp.</li></ul> | Logs stock transfer movement between inventory warehouses. |
| **`createTrip`** | **Collaboration DB** | <ul><li>**`mass`**: Stores trip/shipment record (ID, carrier/vehicle parent ID, type = trip, quantity/value, and `{ref, driver, driverId}` in JSON `data` ‚Äî read by render & `advanceTrip`).</li><li>**`motion`**: Appends audit record (opcode `401` - DISPATCHED); payload carries `{ref}` (used for the Activity desc) but no longer mirrors `driver`.</li></ul> | Dispatches SCM shipment/trip with status logging. |
| **`assignDriver`** | **Collaboration DB** | <ul><li>**`motion`**: Appends status record (opcode `403` - DRIVER_ASSIGNED) containing the assigned driver's entity ID in the payload.</li></ul> | Assigns driver/person entity to a carrier trip. |
| **`advanceTrip`** | **Collaboration DB** | <ul><li>**`motion`**: Performs in-place `phaseUpdateMotion` update (rewrites action opcode to `401` DISPATCHED, `402` IN_TRANSIT, or `109` DELIVERED).</li></ul> | Advances trip status sequentially through lifecycle. |
| **`updateTripEta`** | **Collaboration DB** | <ul><li>**`motion`**: Performs in-place `phaseUpdateMotion` update (updates action opcode `404` - ETA_UPDATE and updates delta/ETA values).</li></ul> | Updates shipping arrival duration estimates. |
| **`logDeliveryAttempt`** | **Collaboration DB** | <ul><li>**`motion`**: Performs in-place `phaseUpdateMotion` update (updates action opcode `410` - DELIVERY_FAILED and details in metadata).</li></ul> | Records failed SCM shipping delivery attempts. |
| **`logReturnRequest`** | **Collaboration DB** | <ul><li>**`motion`**: Performs in-place `phaseUpdateMotion` update (updates action opcode `407` - RETURNED and details in metadata).</li></ul> | Registers SCM returns. |
| **`createProduct`** | **Collaboration DB** | <ul><li>**`matter`**: Inserts product blueprint metadata (ID, code, type = product, title, owner, and pricing JSON in `data`).</li></ul> | Inserts retail product blueprint into `matter` and runs vector store indexing. |
| **`createVariant`** | **Collaboration DB** | <ul><li>**`mass`**: Stores variant item (ID, product parent ID, type = variant, initial stock qty, base price in value, and `{label}` in JSON `data` ‚Äî the authoritative variant label).</li><li>**`motion`**: Appends initial audit record (opcode `406` - TRANSFER_IN) with initial stock quantity delta; payload carries only `{src}`, not a copy of the label.</li></ul> | Creates retail variant with initial stock. |
| **`adjustVariantStock`** | **Collaboration DB** | <ul><li>**`mass`**: Updates the `qty` value (adds or subtracts 10).</li><li>**`motion`**: Appends audit record (opcode `406` - TRANSFER_IN or `405` - TRANSFER_OUT) with the adjustment delta; empty payload ‚Äî opcode + delta convey the movement.</li></ul> | Restocks (+10) or writes off (-10) variant quantities. |
| **`createModifier`** | **Collaboration DB** | <ul><li>**`matter`**: Inserts modifier product metadata (ID, type = product).</li><li>**`relation`**: Links modifier to parent product (src = product, tgt = modifier, type = modifier_of).</li><li>**`mass`**: Stores pricing metadata (ID, matter = modifier ID, type = modifier, price in value, active = 1).</li></ul> | Creates catalog modifier. |
| **`deactivateModifier`** | **Collaboration DB** | <ul><li>**`mass`**: Updates the modifier row's state flag (`active = 0`).</li></ul> | Deactivates variant modifier. |
| **`publishToProfile`** | **Collaboration DB** | <ul><li>**`relation`**: Inserts link row (src = product ID, tgt = profile ID, type = published_to).</li></ul> | Links retail product blueprint to storefront profile. |
| **`unpublishFromProfile`**| **Collaboration DB** | <ul><li>**`relation`**: Deletes the link row matching the product and profile ID.</li></ul> | Unpublishes a product from a storefront profile. |
| **`createLead`** | **Collaboration DB** | <ul><li>**`mass`**: Stores lead record (ID, customer ID, type = lead, projected deal value, and data payload).</li><li>**`motion`**: Appends initial audit record (opcode `303` - NEW_LEAD) with active status.</li></ul> | Registers a CRM lead. |
| **`advanceLead`** | **Collaboration DB** | <ul><li>**`motion`**: Performs in-place `phaseUpdateMotion` update (updates action opcode to `303` NEW, `304` CONTACTED, or `305` CONVERTED).</li></ul> | Moves CRM lead lifecycle stage. |
| **`closeLead`** | **Collaboration DB** | <ul><li>**`mass`**: Updates lead status flag (`active = 0`).</li><li>**`motion`**: Appends audit record (opcode `303` - NEW_LEAD, stage = CLOSED).</li></ul> | Deactivates CRM lead. |
| **`logQuickEvent`** | **Collaboration DB** | <ul><li>**`motion`**: Appends transaction log (opcode `301` - VISIT, `302` - REVIEW with rating/delta, or `309` - OFFER).</li></ul> | Logs interactions directly to `motion` ledger. |
| **`createTicket`** | **Collaboration DB** | <ul><li>**`mass`**: Stores ticket metadata (ID, customer ID, type = ticket, active = 1).</li><li>**`motion`**: Appends initial audit record (opcode `306` - TICKET_OPEN) with subject details in metadata.</li></ul> | Opens support ticket under a customer. |
| **`submitReply`** | **Collaboration DB** | <ul><li>**`motion`**: Appends reply record (opcode `307` - REPLIED) capturing text message inside metadata.</li></ul> | Appends customer/agent conversation reply text. |
| **`resolveTicket`** | **Collaboration DB** | <ul><li>**`mass`**: Updates ticket status flag (`active = 0`).</li><li>**`motion`**: Performs in-place `phaseUpdateMotion` update (rewrites opcode to `308` - COMPLETED).</li></ul> | Resolves support ticket. |
| **`createTask`** | **Private DB** (personal)<br>**Collaboration DB** (shared) | <ul><li>**`matter`**: Inserts task item (ID, type = task, title, description in data).</li><li>**`relation`**: Inserts assignment link if shared (src = customer ID, tgt = task ID, type = task).</li><li>**`mass`**: Inserts due date slot (ID, matter = task ID, type = slot, start/end dates, active = 1).</li></ul> | Inserts task metadata and due date slot. No `motion` row ‚Äî task state lives entirely in the `mass` slot (`active`/`end`). |
| **`toggleTask`** | **Private DB** (personal)<br>**Collaboration DB** (shared) | <ul><li>**`mass`**: Updates task slot status flag (`active = 0` + `end` timestamp if completed, `active = 1` if reopened).</li></ul> | Marks task complete/incomplete. No `motion` row ‚Äî completion state and timestamp are read from the `mass` slot. |
| **`createNote`** | **Private DB** (personal)<br>**Collaboration DB** (shared) | <ul><li>**`matter`**: Inserts note metadata (ID, type = note, text details in data).</li><li>**`relation`**: Inserts context link if shared (src = customer ID, tgt = note ID, type = note).</li></ul> | Inserts note text. Indexes text in vector store. |
| **`createSlot`** | **Collaboration DB** | <ul><li>**`mass`**: Inserts slot item (ID, team member ID, type = slot, start/end times, active = 1, title in data).</li></ul> | Schedules shifts/chores on team/family calendar. No `motion` row ‚Äî the slot is fully described by the `mass` row. |
| **`createBudgetAllocation`**| **Collaboration DB** | <ul><li>**`mass`**: Inserts budget item (ID, account ID, type = budget, limit value, remaining balance in qty, active = 1).</li><li>**`motion`**: Appends initial audit log (opcode `806` - GENERAL_EXPENSE).</li></ul> | Allocates financial budget pools. |
| **`recordExpense`** | **Collaboration DB** | <ul><li>**`mass`**: Updates remaining balance (decreases `qty` by expense amount).</li><li>**`motion`**: Appends expense log (opcode `806` - GENERAL_EXPENSE) with the negative delta and description.</li></ul> | Deducts expense from allocated budget pool. |
| **`createInvoice`** | **Collaboration DB** | <ul><li>**`mass`**: Inserts invoice item (ID, customer ID, type = invoice, total amount in value, active = 1).</li><li>**`motion`**: Appends initial audit log (opcode `801` - PAYMENT_INIT).</li></ul> | Generates check-out invoice. |
| **`resolveInvoiceSuccess`** | **Collaboration DB** | <ul><li>**`mass`**: Updates invoice status flag (`active = 0`).</li><li>**`motion`**: Performs in-place `phaseUpdateMotion` update (rewrites action opcode to `802` - PAYMENT_SUCCESS).</li></ul> | Approves invoice payment. |
| **`resolveInvoiceFailure`** | **Collaboration DB** | <ul><li>**`mass`**: Updates invoice status flag (`active = 0`).</li><li>**`motion`**: Performs in-place `phaseUpdateMotion` update (rewrites action opcode to `805` - PAYMENT_FAIL).</li></ul> | Rejects/cancels invoice payment. |

> [!NOTE]
> All tasks and notes created under the `"general_personal"` customer ID route directly to the offline private DB (`user_private_${self_id}.db`). All other entries route to the synced collaborative databases.

> [!NOTE]
> **Status lives in the opcodes.** A `motion` row's status is conveyed entirely by its `action`/`phase` integer columns ‚Äî `appendMotion`/`phaseUpdateMotion` no longer mirror a `status` string into `motion.data`. On read, the human label is derived from `OPCODE_LABELS[phase || action]` (e.g. `406` ‚Üí "TRANSFER IN"). The `data` column carries only real payload (`src`, `ref`, `reason`, `rating`, phase-history `ph`, etc.). Legacy rows that still contain a `status` field are ignored on read.
