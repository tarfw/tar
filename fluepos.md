### 3.1 POS Master Skill

```markdown
<!-- Generated by LLM -->
<!-- Saved as: form.type='skill', id='skill-pos' -->

---
name: point-of-sale
description: How to manage retail point of sale operations
---

# Point of Sale (POS)

## Core Concepts

### Store
Physical or virtual location where sales happen.
- Has registers (POS terminals)
- Has inventory (stock)
- Has staff (cashiers)
- Has tax rules
- Has hours

### Register
POS terminal where transactions are processed.
- Hardware: printer, scanner, cash drawer
- Assigned to cashier during shift
- Opens/closes shift (cash drawer)
- Logs all transactions

### Product
Item available for sale.
- Can have variants (size, color, etc.)
- Has price (with optional discounts)
- Has stock per location
- Has SKU/barcode

### Order
Single transaction (sale).
- Contains line items (products)
- Calculates: subtotal → tax → discount → total
- Links to payment(s)
- Prints/emails receipt
- Updates inventory

### Payment
Money received for an order.
- Methods: cash, card, mobile wallet, gift card
- Processors: Stripe, Square, manual
- Status: pending → captured → refunded | failed

### Shift
Cashier work period at a register.
- Opens with starting cash
- Tracks sales during shift
- Closes with ending cash + reconciliation
- Variance report (expected vs actual)

## Common Operations

### Open Shift
1. Cashier logs in to register
2. Count starting cash
3. Create shift matter (status=open)
4. Log to motion

### Ring Sale (Checkout)
1. Scan items (or manual entry)
2. Apply discounts (if any)
3. Calculate total (subtotal + tax)
4. Take payment
5. Update inventory
6. Print/email receipt
7. Log everything to motion

### Close Shift
1. Count ending cash
2. Calculate expected (starting + sales - refunds)
3. Variance (actual vs expected)
4. Mark shift closed
5. Generate Z-report

### Restock
1. Receive shipment
2. Update stock levels (attr)
3. If low → trigger reorder workflow

### Refund
1. Find original order
2. Create refund matter
3. Reverse payment
4. Restock items
5. Log to motion

## Best Practices

### Inventory
- Track stock per location
- Set reorder points
- Alert when low
- Audit weekly

### Pricing
- Base price on product
- Discounts via form.type='discount'
- Tax via form.type='tax-rule'
- Never hardcode in code

### Receipts
- Always print/email
- Include: items, totals, payment, refund policy
- Log to motion for audit

### Security
- Each shift is its own matter
- Cashier logged per transaction
- Variance tracking
- Audit trail in motion (append-only)

## Typical Pipeline Stages

### Order
1. Created
2. Items added
3. Total calculated
4. Payment captured
5. Completed (or Failed/Refunded)

### Shift
1. Opening (count starting cash)
2. Active (processing sales)
3. Closing (count ending cash)
4. Closed (reconciled)

## Integration Points

### Payment Processors
- Stripe (cards)
- Square (cards + mobile)
- Manual (cash drawer only)

### Hardware
- Receipt printers
- Barcode scanners
- Cash drawers
- Card readers

### Reporting
- Daily sales (workflow)
- Inventory levels (query)
- Tax reports (workflow)
- Shift reports (motion aggregate)

## Common Skills to Addately

### Per-Tenant
- Their specific products
- Their pricing strategy
- Their tax jurisdictions
- Their payment processors
- Their receipt templates
```

### 3.4 Composite Actions

```typescript
// LLM generates this code → saves to src/actions/composite/checkout.ts
import { defineAction } from '@flue/runtime';
import actionAdvanceStage from '../core/advance_stage';
import actionNotify from '../core/notify';
import actionCalculateTax from '../core/calculate_tax';
import toolCreateMatter from '../../tools/core/create_matter';
import toolUpdateMatter from '../../tools/core/update_matter';
import toolSetAttr from '../../tools/core/set_attr';
import toolLinkGraph from '../../tools/core/link_graph';
import toolAppendMotion from '../../tools/core/append_motion';

export const actionCheckout = defineAction({
  async run({ harness, input }) {
    const { storeId, registerId, cashierId, items, paymentMethod, customerId } = input;
    
    // 1. Validate inputs
    if (!items || items.length === 0) {
      throw new Error('No items in cart');
    }
    
    // 2. Create order
    const orderId = `order-${Date.now()}`;
    let subtotal = 0;
    
    await harness.tools.call(toolCreateMatter, {
      id: orderId,
      form: 'pipeline-order',
      type: 'order',
      scope: storeId,
      title: `Order #${orderId}`,
      data: {
        register: registerId,
        cashier: cashierId,
        customer: customerId || null,
        items: [],
        status: 'created',
      },
      mark: 1,
    });
    
    // 3. Add line items + calculate subtotal
    const lineItems = [];
    for (const item of items) {
      const product = await harness.tools.call(toolGetMatter, { id: item.productId });
      const lineTotal = product.value * item.qty;
      subtotal += lineTotal;
      
      await harness.tools.call(toolLinkGraph, {
        src: orderId,
        rel: 'contains',
        tgt: item.productId,
        data: { qty: item.qty, price: product.value, lineTotal },
      });
      
      lineItems.push({
        product: item.productId,
        qty: item.qty,
        price: product.value,
        lineTotal,
      });
    }
    
    // 4. Apply tax
    const taxResult = await harness.actions.call(actionCalculateTax, {
      storeId,
      subtotal,
    });
    
    const tax = taxResult.taxAmount;
    const total = subtotal + tax - (input.discount || 0);
    
    // 5. Update order with totals
    await harness.tools.call(toolUpdateMatter, {
      id: orderId,
      data: {
        items: lineItems,
        subtotal,
        tax,
        discount: input.discount || 0,
        total,
      },
      mark: 3, // total-calculated
    });
    
    // 6. Process payment
    const paymentId = `payment-${Date.now()}`;
    await harness.tools.call(toolCreateMatter, {
      id: paymentId,
      form: 'form-payment',
      type: 'payment',
      scope: storeId,
      title: `Payment for ${orderId}`,
      value: total,
      data: {
        order: orderId,
        method: paymentMethod,
        status: 'pending',
      },
    });
    
    // 7. Call payment processor (Stripe, Square, etc.)
    const paymentResult = await harness.tools.call(toolPaymentProcessor, {
      paymentId,
      method: paymentMethod,
      amount: total,
    });
    
    if (!paymentResult.success) {
      // Payment failed
      await harness.tools.call(toolUpdateMatter, {
        id: orderId,
        mark: 7, // failed
      });
      
      await harness.tools.call(toolAppendMotion, {
        stream: `matter-${orderId}`,
        seq: Date.now(),
        action: 99993,
        data: { event: 'payment_failed', reason: paymentResult.error },
      });
      
      throw new Error(`Payment failed: ${paymentResult.error}`);
    }
    
    // 8. Mark payment captured
    await harness.tools.call(toolUpdateMatter, {
      id: paymentId,
      data: { status: 'captured', processorRef: paymentResult.ref },
    });
    
    // 9. Update inventory
    for (const item of items) {
      await harness.tools.call(toolSetAttr, {
        matterId: item.productId,
        key: `stock-${storeId}`,
        numDelta: -item.qty,
      });
      
      // Check reorder point
      const stock = await harness.tools.call(toolGetAttr, {
        matterId: item.productId,
        key: `stock-${storeId}`,
      });
      
      const reorderPoint = await harness.tools.call(toolGetAttr, {
        matterId: item.productId,
        key: 'reorder_point',
      });
      
      if (stock.num < reorderPoint.num) {
        await harness.actions.call(actionNotify, {
          to: 'store-manager',
          channel: 'email',
          template: 'restock-needed',
          data: { product: item.productId, stock: stock.num },
        });
      }
    }
    
    // 10. Advance order to completed
    await harness.actions.call(actionAdvanceStage, {
      matterId: orderId,
      targetPhase: 5,
      actionCode: 80013,
    });
    
    // 11. Print/email receipt
    if (customerId) {
      await harness.actions.call(actionNotify, {
        to: customerId,
        channel: 'email',
        template: 'order-receipt',
        data: { orderId, paymentId, items: lineItems, total },
      });
    }
    
    // 12. Log to motion
    await harness.tools.call(toolAppendMotion, {
      stream: `matter-${orderId}`,
      seq: Date.now(),
      action: 99993,
      data: {
        event: 'checkout_completed',
        total,
        paymentId,
        cashier: cashierId,
      },
    });
    
    return {
      orderId,
      paymentId,
      subtotal,
      tax,
      total,
      success: true,
    };
  },
});
```
